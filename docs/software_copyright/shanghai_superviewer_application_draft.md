## 三、软件概述

`SuperViewer` 是一款面向鸟类摄影和照片后期筛选场景的桌面图形软件，用于读取和浏览鸟类照片及其元数据，联动 `慧眼选鸟` 处理结果中的 `report.db` 数据，实现照片预览、EXIF 信息查看、标题/描述编辑、焦点辅助显示、目录过滤筛选和与其他工具之间的文件联动。

该软件采用 Python 语言开发，以 PyQt6 构建桌面界面，主要面向 Windows 64 位与 macOS 双平台运行。软件强调本地化处理、批量浏览效率、摄影元数据可视化和针对鸟类摄影工作流的快速操作能力。

## 四、开发目的

本软件的开发目标主要包括以下几个方面：

1. 为经过 `慧眼选鸟` 处理的照片目录提供更高效的二次浏览、筛选与编辑能力。
2. 在不依赖复杂后期软件流程的前提下，快速查看照片的 EXIF、焦点、标题和描述等核心信息。
3. 建立从筛选、检查、元数据修正到发送至后续切图工具的连续工作流。
4. 提供对 RAW、JPEG、TIFF、HEIC、HEIF 等常见摄影格式的统一浏览能力。
5. 支持中文元数据显示、编辑和写回，降低鸟类摄影用户的操作门槛。

## 五、主要功能说明

### 1. 目录级照片浏览与结果集加载

软件能够从用户选择的目录中读取图像文件，同时识别 `慧眼选鸟` 生成的 `report.db` 数据，建立目录与结果集的对应关系。用户可按目录层级快速查看同一批次或同一路径下的照片数据，而无需手工逐一整理文件。

### 2. 多格式图像预览

软件支持 JPEG、PNG、TIFF 等常见位图格式，也支持 RAW、HEIC、HEIF 等摄影场景下常用格式。预览区域支持缩放显示，并允许在图片切换时保留适当的查看状态，以提高大量照片连续审阅的效率。

### 3. EXIF 与元数据读取显示

软件能够读取照片文件中的 EXIF 信息、XMP 辅助信息及数据库补充数据，在界面中按可读顺序展示诸如拍摄时间、机身、镜头、快门、光圈、ISO、标题、描述等信息。用户可以按需求自定义标签显示顺序，并对不关心的标签进行隐藏。

### 4. 标题与描述编辑写回

软件支持直接在图形界面中双击编辑照片标题和描述，并将修改写回相应元数据存储位置。该功能适用于在图片筛选后对中文鸟名、说明信息进行补充或校正，提高后续输出和归档的规范性。

### 5. 焦点框与构图辅助线显示

软件具备对焦点辅助显示能力，可依据照片 metadata 或关联数据计算并显示焦点框位置。预览区域还支持构图辅助线显示，包括九宫格、黄金分割、对角线等模式，并支持线宽切换，用于辅助摄影构图分析和复核。

### 6. 条件筛选与排序

软件支持按照星级、文件名、精选标记等条件对照片进行筛选，也支持结果列表排序，便于用户从大量照片中快速找出需要优先处理的内容。

### 7. 数据修正与路径一致性维护

软件可处理实际文件路径与 `report.db` 中记录路径不一致的情况，并在符合条件时修正数据库中的路径信息。该能力用于降低跨磁盘、目录移动或工作区调整后带来的数据不一致问题。

### 8. 跨应用发送与联动

软件支持将当前文件发送到其他应用，尤其是发送到 `SuperBirdStamp` 切图工具，形成浏览筛选到模板导出的一体化流程。

### 9. 中文化显示与用户配置保存

软件支持中文标签显示、配置持久化和用户选项管理，能够保存最近目录、标签显示规则、预览辅助线设置等信息，提高连续使用体验。

## 六、技术实现与系统特点

### 1. 总体架构

`SuperViewer` 采用桌面客户端架构，核心由以下部分组成：

- 图形界面层：基于 `PyQt6`，负责主窗口、文件列表、预览区、EXIF 表格、对话框等交互界面。
- 元数据处理层：负责 EXIF、XMP、数据库行数据与界面展示数据之间的转换与合并。
- 预览与焦点处理层：负责图片加载、缩放显示、焦点框计算、构图辅助线叠加。
- 配置与路径管理层：负责用户配置文件、应用名称、窗口标题、平台路径和跨应用通信。
- 共享能力层：通过 `app_common` 提供预览画布、发送到其他应用、日志、元数据处理等共享模块。

### 2. 技术栈

结合仓库当前代码与依赖文件，本软件主要技术栈如下：

- 编程语言：Python 3
- 图形界面：PyQt6
- 图像处理：Pillow、rawpy、pillow-heif
- 元数据读取/写回：ExifRead、piexif、ExifTool 兼容链路
- 数据缓存与工作流衔接：`report.db` 结果缓存、XMP 辅助数据
- 打包方式：PyInstaller

### 3. 跨平台能力

软件开发时考虑了 Windows 64 位与 macOS 的双平台兼容性，主要体现在：

- 使用 `os.path`、`pathlib` 等机制处理路径差异。
- 对打包运行时的工作目录、应用名称、平台标识进行分别适配。
- 在 Windows 平台处理 AppUserModelID，在 macOS 平台处理应用进程名和 Bundle 名称。
- 对外部进程调用、资源路径和发送到其他应用的行为尽量避免写死平台相关实现。

### 4. 性能与可用性设计

为了提升大目录照片浏览效率，软件采用了列表加载、焦点缓存、预览状态缓存与异步预加载机制。对于需要在用户切换照片时实时显示的焦点框和预览图，软件优先复用缓存结果，减少重复解码和重复计算。

### 5. 适配摄影场景的专用能力

本软件并非通用图片查看器，而是面向鸟类摄影后处理流程做了专门设计，包括：

- 识别 `慧眼选鸟` 结果目录与数据库语义；
- 使用摄影 metadata 和焦点信息进行可视化；
- 支持鸟名相关标题、描述的快速编辑；
- 支持向后续鸟类照片模板工具发送文件；
- 保留与构图线、焦点框等摄影复核相关的专业交互。

## 七、运行环境说明

### 1. 硬件环境

- 64 位通用个人计算机
- 建议内存 8GB 及以上
- 建议具备 SSD 存储，用于提升大批量图片浏览速度

### 2. 软件环境

- Windows 10/11 64 位，或 macOS 近年版本
- Python 3.10 及以上开发环境
- PyQt6 图形库
- Pillow、rawpy、pillow-heif、ExifRead、piexif 等图像与元数据处理依赖

### 3. 输入与输出对象

- 输入对象：照片目录、单张照片文件、`report.db`、XMP 辅助文件
- 输出对象：写回后的元数据、更新后的工作状态、发送到外部工具的文件路径信息

## 八、用户操作流程说明

### 1. 启动软件

用户启动软件后，系统显示主窗口，包含文件列表区域、图片预览区域和 EXIF 信息区域。

### 2. 打开目录或文件

用户可选择图片目录，软件自动扫描目录中的照片文件，并尝试识别相关 `report.db` 数据；也可接收外部应用传入的照片路径。

### 3. 浏览与筛选

用户在文件列表中切换照片时，预览区同步显示图片内容，信息区同步显示相应的 EXIF 或数据库字段。用户可使用筛选条件快速缩小待处理范围。

### 4. 查看焦点与构图辅助信息

用户可勾选“显示对焦点”，查看焦点框；也可选择构图辅助线类型和线宽，用于辅助判断构图与主体位置。

### 5. 编辑元数据

用户可直接编辑标题、描述等信息，软件将相应结果写回元数据存储位置，并同步当前会话缓存。

### 6. 发送至其他工具

用户在完成浏览和信息修正后，可将当前照片发送到其他工具继续处理，例如发送至 `SuperBirdStamp` 进行模板化裁切与导出。

## 九、数据处理与文件组织说明

### 1. 数据来源

软件读取的数据主要包括：

- 原始照片文件本身的 EXIF 数据；
- XMP 辅助文件中的补充元数据；
- `慧眼选鸟` 的 `report.db` 缓存结果；
- 用户在本软件中直接编辑后写回的标题、描述等字段。

### 2. 关键数据处理逻辑

软件在展示数据时，会综合文件实际存在路径、数据库缓存路径与辅助文件路径进行解析，以减少因路径变化导致的数据失配。对部分数据来源不完整的场景，软件会采取数据库回退、辅助信息合并或缓存复用方式，提高结果完整性。

### 3. 编码与中文兼容

软件面向中文用户使用，在元数据展示与编辑过程中，重点关注中文鸟名、标题、描述的可读性和写回一致性，避免出现乱码、截断或字段展示不统一的问题。

## 十、原创性与技术特点说明

本软件的原创性主要体现在以下方面：

1. 针对鸟类摄影工作流进行专门设计，围绕 `慧眼选鸟` 结果复核、二次筛选和元数据修正构建完整界面流程。
2. 将图片预览、EXIF 展示、焦点框可视化、构图辅助线和元数据编辑融合在同一桌面软件中，面向摄影筛片需求提供专用交互。
3. 实现了目录级结果读取、数据库与文件路径对齐、元数据写回和跨工具发送的一体化链路。
4. 对 RAW、HEIF、JPEG 等多格式图像进行统一处理，减少摄影素材格式差异对工作流的影响。
5. 在跨平台桌面环境中实现面向中文用户的元数据可视化和编辑能力，具有较强的实用性和行业针对性。
